# Objective:
Find out how to send a patch with a USP email.
While this task may seem simple, the process of sending an email via terminal with a USP account is quite troublesome since the USP email requires authentication on a specific usp page.

# Problems:
I actually had a lot of problems with this tutorial. It was, by far, the longest tutorial in this course for me. And the reason was that I simply could not make Option A (setting up the authentication process with git credential) work. Apparently, this is a common problem, which has a common solution: just giving up on trying to use Option A and just going with Option B (sending the email via docker), which is what I did, and it worked perfectly.


I spent maybe three hours trying to figure out Option A, checking dependencies, changing the config file etc., and maybe 10 minutes with Option B until it worked (haha). It feels to me that Option B should be Option A not the other way around.

# About Tutorial 5:
Tutorial 5 is unnecessary since Tutorial 6 already covers all the steps for sending a patch by email with git (and more since tutorial 5 doesn't explain how to send patches with a USP email).
